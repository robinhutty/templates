NOTE: some of these sections will likely not be required/relevant to any particular incident. Remember that this template is a guide to assist; it is not intended as rigidly prescriptive.

## Incident Summary

A brief summary of the incident.
Include just enough detail about the scope so that the reader can determine whether to engage more with this incident and its retrospective.

### Impact

* Describe the consequences for other systems/users. Consider including what was NOT impacted.

## Incident description

* Describe WHAT happened.

## Detection

* Describe/link how detection occurred by automated systems and/or humans.
* Notifications to <list of roles/teams/individuals>.

## Response

* Who did what? Include reasoning.
* Note obstacles that impeded recovery.

## Recovery

* What were the criteria for calling the incident over/recovered?

## Lessons

* What can we learn about the system(s)/software/processes? Consider using contributing factors and/or "N whys".
* Note previous similar incidents.


## Corrective Actions

* Describe (perhaps with links to tickets) development work or other changes that could/would prevent future occurrence of this (category of) incident.
